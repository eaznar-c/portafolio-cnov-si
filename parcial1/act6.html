Introducción

A través de esta práctica se configuraron dispositivos Cisco para establecer un túnel seguro sitio a sitio, aplicando conceptos fundamentales de ciberseguridad como criptografía simétrica, autenticación mediante claves pre-compartidas, políticas ISAKMP/IKE y listas de acceso para la definición del tráfico interesante.

Esta actividad permitió comprender el funcionamiento práctico de los servicios de seguridad en redes, reforzando los conocimientos teóricos sobre protección de datos en tránsito y gestión de comunicaciones seguras en infraestructuras empresariales.

Procedimiento

  1. Configuración inicial de equipo.
  2. Activar paquete de seguridad (licencia habilitada).
  3. Implementación de ACLs.
  4. Phase 01: ISAKMP Policy.
  5. Phase 02: IPsec Transform Set.
  6. Crear el mapa criptográfico.
  7. Aplicar el mapa criptográfico.

Configuración de Routers

Se configuraron las redes de los routers (Router 1 y Router 3) con sus respectivas direcciones IP utilizando comandos como:

configure terminal ip address X.X.X.X X.X.X.X

Cada router fue configurado con su respectiva IP para establecer la conectividad necesaria antes de implementar el túnel IPsec.

Políticas de Seguridad ISAKMP

ISAKMP (Internet Security Association and Key Management Protocol) es el protocolo encargado de autenticar los dispositivos y establecer un canal seguro. Esto corresponde a la Fase 1 de IKE, donde se crea el canal seguro antes de establecer el túnel IPsec.

Para poder utilizar las políticas ISAKMP en Packet Tracer, fue necesario habilitar la licencia de seguridad del router (C1900), reiniciando el equipo para cargar los comandos necesarios.

Se utilizó la Política 10 para definir los parámetros de seguridad:

crypto isakmp policy 10 encryption aes 256 authentication pre-share

Estos parámetros definen el cifrado y el método de autenticación que se utilizarán para establecer el canal seguro entre los routers.

La configuración se realizó en ambos routers (Router 1 y Router 3) para poder establecer correctamente el túnel VPN entre ambas redes.

Conclusión

La implementación de una VPN IPsec en Cisco Packet Tracer permitió comprender de manera práctica cómo se establecen comunicaciones seguras entre redes remotas a través de una infraestructura pública.

Mediante la configuración de políticas ISAKMP/IKE, claves pre-compartidas, transform sets y listas de acceso para definir el tráfico interesante, se aplicaron conceptos fundamentales de ciberseguridad relacionados con la confidencialidad, integridad y autenticación de la información.
